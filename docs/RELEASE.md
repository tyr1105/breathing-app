# 发布检查清单

## 代码检查

- [ ] TypeScript 编译通过
- [ ] ESLint 无错误
- [ ] 所有导入正确
- [ ] 无 console.error
- [ ] 无未使用的代码

## 功能检查

- [ ] 训练流程完整
- [ ] 声音正常
- [ ] 振动正常
- [ ] 历史记录正常
- [ ] 成就系统正常
- [ ] 分享功能正常
- [ ] 设置保存正常

## 构建检查

- [ ] Web 构建成功
- [ ] APK 构建成功
- [ ] 文件大小合理
  - [ ] HTML < 5KB
  - [ ] CSS < 20KB
  - [ ] JS < 200KB

## 文档检查

- [ ] README.md 更新
- [ ] CHANGELOG 更新
- [ ] 版本号正确
- [ ] 文档完整

## 测试检查

- [ ] 桌面浏览器测试
- [ ] 移动端测试
- [ ] 离线测试
- [ ] 性能测试

## 发布流程

1. 更新版本号 (version.ts)
2. 更新 CHANGELOG
3. 提交代码
4. 等待 CI 构建
5. 下载 APK
6. 安装测试
7. 发布到应用商店（可选）

## 回滚计划

如果发现严重问题：
1. 回退到上一个稳定版本
2. 创建 hotfix 分支
3. 修复问题
4. 重新发布
