## 1. 背景与目标

### 1.1 背景
Wim Hof（冰人）呼吸法是一种经过科学验证的呼吸技巧，可以帮助用户：
- 提升能量和专注力
- 减轻压力和焦虑
- 增强免疫力
- 提高耐寒能力

目前市场上的呼吸训练应用大多功能复杂、操作繁琐，用户闭眼练习时难以获得良好的引导体验。

### 1.2 目标
打造一款**极简、专注、闭眼友好**的冰人呼吸法训练应用：
- **核心目标**: 让用户能够轻松完成标准Wim Hof呼吸训练
- **体验目标**: 闭眼状态下也能顺利完成训练
- **留存目标**: 通过成就系统和数据追踪，激励用户持续练习

### 1.3 成功指标
| 指标 | 目标 | 衡量方式 |
|------|------|----------|
| 完成率 | > 80% | 用户开始训练后完成全流程的比例 |
| 次日留存 | > 40% | 第二天再次打开应用的用户比例 |
| 7日留存 | > 20% | 7天后仍然活跃的用户比例 |
| 平均憋气时长 | > 45秒 | 用户平均憋气时间 |

## 2. 目标用户

### 2.1 用户画像

**主要用户**: 25-40岁健康意识强的都市白领
- **特征**: 工作压力大、追求效率、关注健康
- **场景**: 早晨起床后、午休时、睡前
- **痛点**: 现有应用太复杂，闭眼练习时不知道什么时候吸气呼气

**次要用户**: 对冥想和呼吸练习感兴趣的初学者
- **特征**: 刚接触呼吸练习，需要清晰引导
- **场景**: 随时随地，碎片化时间
- **痛点**: 不知道如何正确练习

### 2.2 用户需求
1. **明确引导**: 知道什么时候吸气、什么时候呼气、什么时候憋气
2. **声音提示**: 闭眼练习时通过声音知道该做什么
3. **进度追踪**: 看到自己的进步，有成就感
4. **简单操作**: 一键开始，不需要复杂设置

## 3. 核心功能

### 3.1 训练流程 (MVP)

**标准训练流程**:
```
开始 → 安全提示 → 第1轮(30次呼吸 → 憋气 → 恢复15秒) 
     → 第2轮(30次呼吸 → 憋气 → 恢复15秒) 
     → 第3轮(30次呼吸 → 憋气 → 恢复15秒) 
     → 完成
```

**呼吸阶段**:
- 深吸气（用户自定时长，默认2秒）
- 自然呼气（用户自定时长，默认1秒）
- 重复30次（可自定义10-50次）

**憋气阶段**:
- 呼气后憋气，计时开始
- 用户感到需要呼吸时点击"恢复呼吸"
- 记录憋气时长

**恢复阶段**:
- 深吸气并憋气15秒（可自定义）
- 倒计时提示
- 进入下一轮或完成

### 3.2 声音与振动

**声音提示** (核心功能 - 闭眼友好):
| 场景 | 声音类型 | 说明 |
|------|----------|------|
| 吸气 | 上升音调 | 300Hz → 500Hz, 0.3秒 |
| 呼气 | 下降音调 | 500Hz → 300Hz, 0.2秒 |
| 开始憋气 | 单音提示 | 600Hz, 0.5秒 |
| 恢复倒计时 | 滴答声 | 800Hz, 0.1秒 |
| 训练完成 | 上升和弦 | C-E-G 三音 |

**振动反馈** (辅助功能):
- 吸气: 短振动 (50ms)
- 呼气: 短振动 (50ms)
- 憋气开始: 长振动 (200ms)
- 训练完成: 三连振动

### 3.3 数据追踪

**训练记录**:
- 每轮憋气时长
- 总憋气时长
- 训练日期时间
- 呼吸次数配置

**统计数据**:
- 连续训练天数
- 总训练次数
- 个人最佳憋气记录
- 总憋气时长累计

**历史记录**:
- 最近100条训练记录
- 7天训练趋势图
- 周报统计

### 3.4 成就系统

**成就类型**:
| 成就 | 条件 | 图标 |
|------|------|------|
| 初次尝试 | 完成第一次训练 | 🎯 |

| 一周坚持 | 连续训练7天 | 🌟 |
| 月度冠军 | 连续训练30天 | 👑 |
| 憋气新手 | 憋气达到30秒 | 💨 |
| 憋气达人 | 憋气达到1分钟 | 💪 |
| 憋气大师 | 憋气达到90秒 | 🏆 |
| 冰人附体 | 憋气达到2分钟 | ❄️ |
| 勤奋学员 | 完成10次训练 | 📚 |
| 呼吸专家 | 完成50次训练 | 🎖️ |

## 5. 不做什么 (Out of Scope)

### V3.0 不包含

1. **社交功能**
   - 好友系统
   - 排行榜
   - 群组训练
   - 原因: 增加复杂度，影响专注体验

2. **付费功能**
   - 高级课程
   - 会员订阅
   - 原因: 保持产品纯粹性

3. **复杂呼吸模式**
   - 4-7-8呼吸法
   - 箱式呼吸
   - 原因: 专注Wim Hof方法

4. **健康数据集成**
   - Apple Health
   - Google Fit
   - 原因: 简化首次发布

5. **视频教程**
   - 在线视频
   - 直播课程
   - 原因: 保持轻量级

### 未来可能添加

- 冥想模式
- 白噪音背景
- 冷水澡计时器
- 社区分享

## 6. 技术方案

### 6.1 技术栈选型

| 类别 | 选择 | 理由 |
|------|------|------|
| 前端框架 | React 18 | 生态成熟，TypeScript支持好 |
| 语言 | TypeScript | 类型安全，减少错误 |
| 样式 | Tailwind CSS | 快速开发，体积小 |
| 动画 | Framer Motion | 声明式，性能好 |
| 状态管理 | useState + Context | 应用简单，不需要Redux |
| 构建 | Vite 7 | 快速，现代化 |
| 移动端 | Capacitor 7 | 同一套代码，原生体验 |
| CI/CD | GitHub Actions | 自动化APK构建 |

### 6.2 数据存储

**LocalStorage**:
- 用户设置
- 训练历史（最多100条）
- 用户统计
- 成就记录

**原因**:
- 简单可靠
- 无需服务器
- 隐私保护

### 6.3 音频方案

**Web Audio API**:
- 生成简单提示音
- 无需外部音频文件
- 低延迟

### 6.4 离线支持

**Service Worker**:
- 缓存核心资源
- 离线可用
- PWA支持
